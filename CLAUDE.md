## ドキュメント管理ルール

本プロジェクトでは以下のドキュメント体系で管理します。各ファイルの役割を理解した上で、適切なファイルに情報を記録してください。

| ファイル | 役割 | 書くべき内容 |
|---|---|---|
| `docs/development.md` | 開発手順書 | 環境構築・コマンド・デプロイ手順など、開発者が手を動かす際に参照する情報 |
| `docs/plan.md` | 実装計画メモ | 決定前のアイデア・選択肢・議論の記録。探索の場として自由に使う |
| `docs/spec.md` | 仕様書 | `plan.md` での議論を経て**確定した**仕様のみ。未確定事項は書かない |
| `docs/todo.md` | TODO リスト | 実装タスクの一覧。完了済みタスクは削除せずチェックを付ける |
| `docs/knowledge.md` | 汎用ナレッジ | プロジェクト固有でない知見。他プロジェクトへ流用できる内容に限定 |

### 運用ルール

1. **plan.md → spec.md の流れを守る**
   - 仕様の検討は `plan.md` で行い、確定したものだけ `spec.md` に記録する。
   - `spec.md` に未確定の内容を書かない。

2. **knowledge.md はプロジェクト非依存に保つ**
   - プロジェクト固有の情報（データソースのURL、モデルのパスなど）は書かない。
   - 汎用的な技術知見・ドメイン知識のみ記録する。

3. **todo.md の完了タスクは残す**
   - 完了したタスクを削除すると履歴が失われる。`- [x]` でチェックを付けて残す。

4. **development.md は実行可能な状態に保つ**
   - 手順が古くなったらすぐ更新する。
   - コマンドは実際に動作するものだけ記載する。
