### Docker コンテナと cron の常時起動

Docker コンテナはメインプロセス（PID 1）が終了すると停止する。  
`crond` はデフォルトでデーモン化（バックグラウンド移行）するため、そのまま CMD に指定するとコンテナがすぐ落ちる。

**解決策: `-f` フラグでフォアグラウンド実行する**

```dockerfile
CMD ["crond", "-f", "-l", "2"]
```

- `-f` … フォアグラウンドで動作し続ける（デーモン化しない）
- `-l 2` … ログレベル（dcron の場合）

**リソースオーバーヘッドはほぼゼロ。**  
`crond` は毎分1回スケジュールを確認してスリープするだけなので、CPU 使用率は実質 0%、メモリも数 MB 程度。フォアグラウンド/バックグラウンドの違いはプロセスの親子関係の話であり、スリープ中の資源消費には影響しない。
