# フェーズ5以降の精度改善案

フェーズ4時点で recovery_rate ≈ 80% に到達。100% 超えを目指した次の手を整理する。

過去フェーズの実験結果は [experiments.md](./experiments.md) を参照。

---

## A. 特徴量追加（フィーチャーエンジニアリング）

### A-1. 出走間隔（前走からの日数）

- 前走日付との差（`days_since_last_race`）を計算して追加
- 連闘・長期休養明けはパフォーマンスに影響することが知られている
- 実装コスト：小（preprocess 内で `shift(1)` した日付との差分）
- 期待効果：中

### A-2. 前走との着差（margin）

- 前走で勝った馬の「大差勝ち」と「ハナ差勝ち」は意味が異なる
- `finishing_margin` を直近3走・5走で平均化
- **注意**: 着差データが DB に格納されているか要確認
- 実装コスト：中（DB スキーマ確認＋rolling 集計）
- 期待効果：中〜高

### A-3. 負担重量のレース内相対値

- `weight_carried` をレース内で正規化（平均との差、または z-score）
- 馬の体格に対して重い斤量かどうかを相対的に表現できる
- 実装コスト：小（groupby + transform）
- 期待効果：小〜中

### A-4. 前走クラス昇降格フラグ

- 前走が今走より上のクラス（格上挑戦）か、下のクラス（降格）かのフラグ
- 格上挑戦失敗馬の次走、降格馬の反動などを捉えられる
- `grade` のラベルエンコード＋ shift(1) との差分で実現可能
- 実装コスト：中
- 期待効果：中

### A-5. 馬体重の変動トレンド

- 直近3走の馬体重の線形トレンド（増加傾向 or 減少傾向）
- 成長途上の若馬（体重増加）と衰え（体重減少）を捉える
- 実装コスト：小（rolling の mean に加えて傾きを計算）
- 期待効果：小

### A-6. 騎手の直近N走ローリング勝率

- 現在は騎手の累積勝率のみ。騎手の「今の調子」（直近10走・30走）を追加
- 調教師も直近30走は実装済みだが、騎手の直近ローリングが未実装
- 実装コスト：小（既存の trainer_win_rate_last30 と同じパターン）
- 期待効果：中

## B. モデリング・学習戦略

### B-1. Optuna によるハイパーパラメータ最適化

- 現在のパラメータはほぼデフォルト（`num_leaves=63`, `lr=0.05`, `num_boost_round=500`）
- Optuna で `num_leaves`, `learning_rate`, `min_child_samples`, `feature_fraction` などを探索
- 実装コスト：中（optuna 導入＋交差検証の設計）
- 期待効果：中（通常 1〜3pp 程度）

### B-2. カスタム損失関数（Focal Loss）

- 現在は `binary_logloss`。hard sample（予測困難な馬）に重点を置く Focal Loss を試す
- 的中しにくい馬（穴馬）への予測精度を高め、回収率改善につなげる
- 勾配・ヘッセを手動実装して LightGBM の custom objective として渡す
- `gamma` パラメータで hard sample への集中度を調整
- 実装コスト：大（custom objective 実装、勾配・ヘッセの導出）
- 期待効果：中（穴馬の的中率向上に寄与する可能性）

### B-3. 確率キャリブレーション（Isotonic Regression / Platt Scaling）

- LightGBM の raw 出力はキャリブレーションされていない場合がある
- レース内での win_prob の正規化はすでに実施済み（model.py）
- 残存バイアスを Isotonic Regression でキャリブレーション
- 実装コスト：中
- 期待効果：小〜中（期待値フィルタの精度が上がる可能性）

### B-4. ランキング学習（LambdaRank / LambdaMART）

- 現在は1馬ごとに独立した binary classification。レース内の馬をまとめてランキング問題として解く
- LightGBM の `objective=lambdarank` または `objective=rank_xendcg` を試す
- 実装コスト：大（データ形式の変更、group パラメータの設定）
- 期待効果：大（レース内相対順位を直接最適化できる）

## C. 評価・分析

### C-1. 特徴量重要度の分析と低寄与特徴量の除去

- `model.feature_importance()` で各特徴量の寄与度を可視化
- 重要度の低い特徴量を除くことでオーバーフィット抑制・予測の安定化が期待できる
- 実装コスト：小
- 期待効果：小〜中

### C-2. 回収率の信頼区間・シャープレシオ計算

- テストデータでの回収率は1つの点推定に過ぎない
- ブートストラップで信頼区間を求め、改善が統計的に有意かを判定する仕組みを追加
- 実装コスト：中
- 期待効果：なし（精度改善ではなく評価の信頼性向上）
