# アンサンブル予報の応用：モンテカルロ着順シミュレーション vs モデルアンサンブル（2026-05-25）

## 発端

「競馬はカオスだと思っていて、気象予報で使うアンサンブル予報の手法を活かせないか」という着想からの議論。

## 気象アンサンブル予報の本質

- 気象予報のアンサンブルは「**物理方程式という決定論的な前進モデル**があり、**初期値をわずかに摂動**させて複数回シミュレートし、カオス的に発散する結果の**ばらつきを確率分布に変換する**」手法。
- 価値の源泉は、決定論的なカオス系を確率予報に変えるところにある。

## 競馬への移植

そのまま移すなら近いのは **モンテカルロでレースを何千回もシミュレートし、各馬の着順分布から勝率を出す** やり方。

- 回収率狙いと相性が良い。EV 計算（[prediction-accuracy-followup.md](./prediction-accuracy-followup.md)）に必要なのは、まさにキャリブレーションされた勝率分布だから。

### 気象との決定的な違い（注意点）

- 気象には摂動をかける「**本物の物理モデル**」があるが、**競馬には無い**。
- 統計モデルの入力を揺らしても、出てくるのは**そのモデル自身の不確実性の幅**であって、ベースモデルに市場（オッズ）が持っていない情報が無い限り、アンサンブル化しても**市場に対するエッジは生まれない**。
- カオスそのものを当てにいくより、**勝率の確率分布を市場よりいかに正確に作るか**が勝負所。

## 検討した2方向

| 方向 | 何をするか | エッジに対する役割 |
|---|---|---|
| (a) モンテカルロ着順シミュレーション | 各馬の能力分布から着順を多数回サンプリングし、券種ごとの確率を導出 | エッジを **増幅** する |
| (b) 複数の異質なモデルのアンサンブル | 異なるモデルを束ねて予測の精度・頑健性を上げる | エッジを **作る／良くする** |

## どちらが効果が高いか（核心）

**この2つは同じ仕事を奪い合っているわけではない**。別軸の手法なので「どっちが上か」は状況次第。

### モンテカルロ ＝ エッジの増幅

- ベースモデルの各馬の能力分布を、**馬券種ごとの整合的な確率分布**に変換する。
- 特に **馬連・三連単などの組合せ馬券**で威力を発揮する。同時確率を解析的に出すのは難しく、市場（控除率込みの公衆）も読み切れていないため、ここはエッジが出やすい。
- 逆に **単勝は MC をかけてもほぼ市場オッズを再現するだけ**になりがち。
- **重要：MC の効果はベースモデルの質で頭打ちになる。** 各馬の能力推定が市場より正確でなければ、綺麗な確率分布を作っても回収率は控除率（JRA なら 20〜25%）を超えられない。**MC はエッジを増幅するだけで、無いエッジは生み出せない。**

### モデルアンサンブル ＝ エッジの創出／改善

- ベース予測そのものの精度・頑健性を上げる。エッジがそもそも有るかどうかを左右する部分に効く。

## 実務的な結論・進め方

**「MC で十分か」はベースモデルにエッジがあるかどうかで決まり、それは今の EV パイプラインで測れる。**

- 既に事前オッズを join して EV 計算する仕組みがある（直近コミット `feat(predictor): load_predict_data で事前オッズを join し EV 計算に使用`）。
- 過去データで **MC 勝率 vs 実オッズの EV** を回して判定する：
  - **単勝 EV ベットが控除率を超えて回収率 > 100% に乗る** → ベースモデルにエッジあり。MC で馬券種を広げる方向が高効率。アンサンブルは後回しで OK。
  - **全然乗らない** → ボトルネックはベースモデルの精度。MC をいくら作り込んでも無駄で、ここが **アンサンブルの出番**。

→ 先に **MC を薄く実装して当たり判定する**のは、実装コストも低く正しい第一歩。MC は実装が簡単な割にレバレッジが高い。
